# scratch

Outil de control du flot d'exécution (backward edge), une idée d'Ernest. A la base il avait vendu une utilisation d'intelCET, 
le client ayant demandé "une sécurité identique à celle du kernel Windows" (depuis quand c'est une référence?).

Après une première étude, on s'est rendu compte que la distrib et le kernel utilisée par SAFE n'était pas forcément compatible, 
puis que la fonction ne serait juste pas exposée par l'hyperviseur (SAFE devrait être virtualisé d'après le DO, Ernest assure 
que "docker ça passe"). D'où cette solution de "ShadowStack logicielle", sortie d'on ne sait où par Ernest. 

# notes techniques:

```
5.3. Utilisation
ShadowGuard Pro est un exécutable qui lance et surveille l'application à sécuriser via ptrace. La syntaxe de base est la suivante :

shadowguard-pro [options]  [arguments]
Exemple :

shadowguard-pro --log-level=debug /usr/bin/mon_application --option1 --option2
5.4. Options Courantes
--log-level : Niveau de journalisation (debug, info, warning, error)
--output : Fichier de sortie pour les logs
--no-protect : Désactive la protection pour le débogage
--help : Affiche l'aide
```

Comment perdre deux jours... Il n'y a absolument pas de commande line, le chemin du binaire protégé (*auth*) doit être configuré 
à la compilation, pour "plus de sécurité". Et comme la startup refuse (logiquement) de fournir leurs sources, un technicien est 
passé configurer et compiler le binaire dans nos locaux. Le mec est super !

Ceci dit ça semble fonctionner, l'impact sur les performances est catastrophique par contre. Heureusement qu'*auth* ne fait rien 
de très complexe.
